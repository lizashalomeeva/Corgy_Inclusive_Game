# 🚀 Розгортання у виробничому середовищі (Production Deployment)

Цей документ призначений для Release Engineer / DevOps фахівців і описує процес збірки та розгортання релізної версії **Corgy Inclusive Game** для кінцевих користувачів.

## 1. Вимоги до апаратного забезпечення (Build Server / CI Runner)
Для автоматизованої збірки проєкту (CI/CD) або ручного релізу потрібен сервер (або локальна машина) з такими мінімальними характеристиками:
* **Архітектура:** x86_64 або ARM64.
* **CPU:** Мінімум 4 ядра (рекомендується 8 для швидкої компіляції Gradle).
* **Оперативна пам'ять (RAM):** Мінімум 8 ГБ (рекомендується 16 ГБ).
* **Дисковий простір:** 50 ГБ вільного місця (SSD) для кешу Gradle та Android SDK.

*Вимоги для кінцевого користувача:* Пристрій на базі Android 8.0 (API level 26) або вище.

## 2. Необхідне програмне забезпечення
На сервері збірки повинні бути встановлені:
* ОС: Ubuntu 22.04 LTS (або аналогічна macOS/Windows).
* **Java Development Kit (JDK):** Версія 17.
* **Android SDK:** Платформа 34, Build-Tools 34.0.0+.
* **Git:** Останньої стабільної версії.

## 3. Налаштування мережі
* Сервер збірки повинен мати вихідний інтернет-доступ для завантаження залежностей з `dl.google.com` та `repo.maven.apache.org`.
* Клієнтський додаток у production-режимі потребує доступу до Leaderboard API: відкриті порти `443` (HTTPS) для домену `lizashalomeeva.github.io`.

## 4. Конфігурація серверів (Керування ключами)
Для створення Production-збірки необхідно налаштувати змінні середовища або GitHub Secrets для цифрового підпису додатку:
* `KEYSTORE_FILE` — шлях до файлу `.jks`.
* `KEY_ALIAS` — псевдонім ключа.
* `KEY_PASSWORD` — пароль від ключа.
* `STORE_PASSWORD` — пароль від сховища ключів.
*(Увага: Ніколи не комітьте файл Keystore у відкритий репозиторій!)*

## 5. Налаштування СУБД
* **Клієнтська частина:** Гра використовує вбудовані механізми ОС Android (SharedPreferences) для локального збереження рекордів та налаштувань інклюзивності. Створення зовнішніх баз даних не вимагається, ОС ініціалізує сховище автоматично при інсталяції APK.
* **Серверна частина:** Використовує безсерверну архітектуру. База даних та API розгортаються автоматично через GitHub Pages.

## 6. Розгортання коду (Збірка та Публікація)
Процес релізу складається з таких кроків:

1. **Очищення кешу:**
   ```bash
   ./gradlew clean
   2. **Запуск тестів:** `./gradlew test`
3. **Генерація Production-збірки (AAB):** `./gradlew bundleRelease`
4. **Публікація:** Згенерований файл `app-release.aab` завантажується у Google Play Console (канал Production).

## 7. Перевірка працездатності (Health Check)
1. **Збірка:** CI/CD пайплайн успішний (статус `SUCCESS`).
2. **API Health:** Запит `curl -I https://lizashalomeeva.github.io/Corgy_Inclusive_Game/api/` повертає `200 OK`.
3. **Моніторинг:** Crash-Free sessions > 99% у консолі розробника після релізу.

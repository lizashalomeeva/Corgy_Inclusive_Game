# Статичний аналіз коду (Linting)

## Обраний інструмент
Для проєкту використовується **Android Lint** та стандартна статична типізація Java. 
Причина вибору: це офіційний, найпотужніший інструмент від Google для Android-розробки, який глибоко інтегрований у систему збірки Gradle і перевіряє не лише стиль, а й доступність (Accessibility), безпеку та продуктивність.

## Базові правила
Конфігурація знаходиться у файлі `app/lint.xml`:
- `HardcodedText`: Забороняє пряме використання тексту в XML. Весь текст має бути у `strings.xml`.
- `ContentDescription`: Вимагає обов'язкового текстового опису для всіх картинок (для скрінрідерів).
- `RtlHardcoded`: Вимагає підтримки розташування тексту справа наліво.

## Статична типізація
Проєкт розроблено мовою Java, яка є строго типізованою. Усі перевірки типів виконуються автоматично компілятором `javac` під час збірки.

## Інтеграція з процесом збірки та запуск
Для комплексної перевірки коду запустіть скрипт:
- На Windows: подвійний клік по `run_lint.bat`
- В терміналі: `./gradlew lint`

## Git Hooks
У проєкті налаштовано `pre-commit` хук. Перед кожним створенням коміту автоматично запускається перевірка лінтером. Якщо знайдено помилки — коміт скасовується до їх виправлення.

# 🔄 Оновлення системи

Цей документ описує процедуру оновлення клієнтського додатка **Corgy Inclusive Game** та серверної частини, а також кроки для відкату у разі критичних збоїв.

## 1. Підготовка до оновлення

* **Створення резервних копій:** * Зробити бекап поточних ключів підпису (`.jks`).
  * Створити тег релізу в Git (наприклад, `git tag v1.0.1` перед початком злиття нової гілки).
* **Перевірка сумісності:** * Перевірити зміни у `build.gradle.kts` (чи не було підвищено `minSdkVersion`, що може відрізати старих гравців).
  * Переконатися, що нова версія Leaderboard API зворотно сумісна з попередніми версіями гри.
* **Планування часу простою:**
  * *Клієнт:* Простій відсутній. Гравці продовжують грати у стару версію, поки не завантажать оновлення з Google Play.
  * *API:* GitHub Pages оновлюється безшовно (Downtime < 5 секунд під час деплою).

## 2. Процес оновлення

1. **Зупинка потрібних служб:** * Не вимагається Serverless архітектура.
2. **Оновлення конфігурацій:**
   * Відкрити `app/build.gradle.kts` і обов'язково збільшити:
     * `versionCode` (наприклад, з 1 на 2).
     * `versionName` (наприклад, з "1.0" на "1.1").
3. **Міграція даних:**
   * Якщо у грі додалися нові налаштування інклюзивності, оновити клас роботи з `SharedPreferences`, додавши значення за замовчуванням для нових ключів, щоб гра не впала у старих користувачів.
4. **Розгортання нового коду:**
   * Зібрати новий AAB файл: `./gradlew bundleRelease`
   * Завантажити новий AAB у Google Play Console. Спочатку рекомендується випустити оновлення на 10-20% користувачів, щоб перевірити стабільність.

## 3. Процедура відкату [+1 бал]

Якщо після оновлення виявлено критичний баг (наприклад, гра вилітає при старті), необхідно виконати відкат:

### Відкат Клієнта
*Увага: Google Play не дозволяє "видалити" нову версію або завантажити старий APK з меншим `versionCode`.*
1. У Google Play Console негайно натиснути **"Halt Rollout"**, щоб зупинити розповсюдження багнутої версії.
2. У терміналі повернутися до стабільного коду (знайти попередній тег/коміт):
   `git checkout tags/v1.0.0`
3. У `build.gradle.kts` збільшити `versionCode` (наприклад, до 3), щоб він був вищим за багнуту версію, але залишити стабільний код.
4. Зібрати новий AAB (`./gradlew bundleRelease`) і терміново випустити його на 100% користувачів.

### Відкат Сервера (Leaderboard API)
1. Знайти хеш стабільного коміту для гілки з документацією.
2. Виконати команду: `git revert <commit_hash>` (або скасувати останній Pull Request).
3. Зробити пуш у гілку `master`. GitHub Actions автоматично перезібере стабільну версію API за кілька хвилин.
